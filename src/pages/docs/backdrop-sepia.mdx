export const classes = {
  plugin: function () {
    return function ({ addUtilities }) {
      addUtilities({
        '.backdrop-sepia-0': { '--tw-backdrop-sepia': 'sepia(0)' },

      })
    }
  }
}

## Usage

Use the `backdrop-sepia` and `backdrop-sepia-0` utilities alongside the `backdrop-filter` utility to whether an element's backdrop should be rendered as sepia or in full color.

```html purple
<template preview>
  <div class="h-full w-full p-4 rounded-2xl bg-purple-200">
    <div class="flex justify-around">
      <div class="flex-1">
        <p class="text-center text-sm text-purple-800 font-medium mb-1">.backdrop-sepia-0</p>
        <div class="mx-auto h-24 w-24 rounded-xl shadow-md backdrop-filter backdrop-sepia-0">
        </div>
      </div>
      <div class="flex-1">
        <p class="text-center text-sm text-purple-800 font-medium mb-1">.backdrop-sepia</p>
        <div class="mx-auto h-24 w-24 rounded-xl shadow-md backdrop-filter backdrop-sepia">
        </div>
      </div>
      <div class="flex-1"></div>
    </div>
  </div>
</template>

<div class="backdrop-filter **backdrop-sepia-0** ..."></div>
<div class="backdrop-filter **backdrop-sepia** ..."></div>
```

## Responsive

To control an element's backdrop sepia filter at a specific breakpoint, add a `{screen}:` prefix to any existing backdrop sepia utility. For example, use `md:backdrop-sepia-0` to apply the `backdrop-sepia-0` utility at only medium screen sizes and above.

```html
<div class="backdrop-filter backdrop-sepia **md:backdrop-sepia-0** ...">
  <!-- ... -->
</div>
```

For more information about Tailwind's responsive design features, check out the [Responsive Design](/docs/responsive-design) documentation.
